# ferray workspace coverage report

Generated by `cargo tarpaulin --workspace --engine llvm`.

Re-run with:
```bash
cargo tarpaulin --workspace --exclude ferray-python \
  --exclude-files 'fuzz/*' --exclude-files 'benchmarks/*' \
  --engine llvm --out Html --out Lcov --output-dir coverage \
  -- --skip kernel_edge_i8_signed_handles_partial --skip force_scalar_env
```

(Two tests are skipped because they fail under tarpaulin's debug build:
`force_scalar_env` asserts `!force_scalar()` which contradicts coverage
runs that need scalar SIMD fallback off; `kernel_edge_i8_signed_handles_partial`
hits a debug-build i8 multiplication overflow that doesn't reproduce in
release. Both are pre-existing test bugs unrelated to coverage.)

## Reading the zeros

Three crates report 0% — none of them are actually untested:

- **`ferray-numpy-interop`**: tests are `#[cfg(feature = "python")]`-gated and need a live Python interpreter to run. Tarpaulin doesn't link libpython, so those tests are silently skipped. Coverage of this crate's logic happens via the ferray-python pytest suite (525/525 passing) which exercises every conversion path.
- **`ferray-python`**: `test = false` in Cargo.toml — wheel-only crate. Coverage happens via the pytest suite under `tests/*.py`.
- **`ferray-core-macros`**: proc-macro crate. tarpaulin doesn't instrument proc-macros directly because they execute at compile time, not at test runtime. The macros' behaviour is exercised transitively by every test in every consumer crate that uses `s!` or `promoted_type!`.

## Reading the underwater crates

- **`ferray-linalg` (61.9%)** — biggest crate by line count. The uncovered portion is largely batched-operation edge paths and the AVX-512 GEMM kernels (which require AVX-512 hardware to execute, and CI runs on AVX2). Filed for follow-up if pushing coverage higher matters.
- **`ferray-polynomial` (62.4%)** — six basis classes with overlapping test coverage. Some of the extras-module conversions (poly2cheb, poly2lag, etc.) lack focused unit tests beyond round-trip property tests in the umbrella suite. Test gaps are tracked under existing issues #245-#255.
- **`ferray-autodiff` (63.2%)** — the surface is small but second-derivative paths and complex-input paths are deferred (issues #303, #304, #305). Of the lines tarpaulin marks uncovered, most are the deferred surfaces.

## Going forward

This is a one-time analysis (issue #44). For ongoing coverage tracking,
the path forward is integrating tarpaulin into CI:

1. Add a `coverage` job to `.github/workflows/ferray-python-wheels.yml`
   (or a sibling workflow for the Rust test surface) that runs the
   tarpaulin command above on Linux.
2. Upload `lcov.info` to coveralls.io or codecov.io for historical
   tracking.
3. Optionally fail PRs that drop coverage by more than X%.

Not done here — out of scope for the analysis-only issue.
